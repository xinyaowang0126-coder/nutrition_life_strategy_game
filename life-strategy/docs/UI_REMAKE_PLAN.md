# UI Remake 后续工作与验收方案

## 目标

把主流程收敛为一套“场景先于脚本”的 Godot UI：主要结构、尺寸、主题、层级和动画入口都能在编辑器中直接查看与调整；脚本只负责加载数据、切换阶段、响应信号和选择响应式布局档位。

界面体验以“背景负责氛围、道具负责入口、卡牌负责选择、便签负责详情”为主。默认画面不堆叠规则文本，鼠标悬停、键盘焦点或触屏信息按钮再展开详细信息。

## 架构约束

- 主入口：`scenes/game_v2/GameRootV2.tscn`，编排七天流程、背景、HUD、提示与模态层。
- 阶段场景：用餐来源、食堂、外卖、便利店、宿舍存粮、行动、睡眠分别保存在 `scenes/game_v2/stages/`，禁止在控制器脚本中从零拼装整页 UI。
- 复用组件：人物状态、紧凑卡牌、便签详情、渐隐提示、每日小结位于 `scenes/game_v2/components/`。
- 视觉规范：共享 `scenes/game_v2/V2Theme.tres`；稳定的字号、色彩、边框和间距优先放入 Theme 或 `.tscn`。
- 脚本职责：`scripts/ui_v2/` 只绑定数据、路由信号、播放反馈和切换桌面/移动端布局档位。动态生成仅限同一组件的内容列表，必须实例化现有 `.tscn` 组件。
- 编辑方式：新增或修改 Godot 节点、资源、信号和场景属性时优先使用编辑器/MCP；文本工具用于数据、测试、文档与小范围脚本逻辑。

- 日期使用完整格式“第 1 天 / 第 7 天”。
- 人物立绘贴底并提高到接近半屏以上，状态板靠近人物且不再占满左栏。
- 状态项使用完整名称与图形符号，并恢复正常、警告、危险的动态颜色；复习值使用独立的中性色。
- 用餐来源改为透明背景的食堂窗口、外卖手机、便利店货架、宿舍存粮道具，并加入阴影、悬停抬升和更醒目的名称。

- 食物饱食度已经下调，来源、行动和食物文案已改成更具体的校园生活口吻。
- V2 控制器已迁移七天循环、餐费、库存、组合、行动、睡眠、每日小结和结局流程。

## 收尾优先级

### P0：完整流程

- 从主菜单开始连续走完“来源 → 选餐 → 行动”三轮、睡眠、每日小结和下一天，确认第七天能进入结局。
- 分别验证食堂拖放/点击入盘、外卖菜单滑动、便利店货架选择和宿舍库存扣减。
- 验证返回、跳过、确认、余额不足、来源未开放、库存耗尽和选择上限等边界。
- 检查所有全屏装饰层的 `mouse_filter`，确保透明层不拦截卡牌与按钮。

### P0：多端与输入

- 桌面基准：1920×1080、1600×900、1280×720。
- 移动端基准：720×1280（9:16）竖屏；检查安全区、最小触控面积、文字换行、弹窗和底部操作区。
- 桌面支持鼠标悬停与点击；触屏使用轻触选中、再次确认或独立确认键，并保留可见的信息按钮。
- 卡牌滑动和点击必须有手势死区，避免拖动列表时误选；键盘焦点需有清晰描边。

### P1：视觉与反馈

- 统一食物、行动、睡眠卡牌的边框语义：类型色、选中金色、禁用灰色、危险/不足红色。
- 隐藏滚动条但保留滑动能力，避免横向白线破坏画面。
- 补齐卡牌确认回弹、数值短暂闪色、时段背景淡入淡出和弹窗出现/离场动画。
- 检查透明道具边缘、阴影偏移和不同宽高比下的完整性；清理无引用的重复素材。
- 评估音效触发点：选中、取消、确认、花费、结算和新一天开始。

### P1：内容与平衡

- 通读全部玩家可见文字，继续删去解释式、总结式和重复提示，只留下具体动作与后果。
- 用固定种子跑多轮七天流程，检查饱食、压力、身体负担和金钱是否仍存在单一最优解。
- 让不同来源的卡池、费用、库存和选择上限形成明确但不过度惩罚的取舍。

## 完成定义

这次重构不是“截图看起来更好”即完成。只有当 V2 成为默认入口、四类用餐交互和完整七天流程都能在桌面与 720×1280（9:16）触屏中操作、状态与结算反馈清楚、自动化测试和编辑器日志无新增错误时，才可以合并到主分支。
