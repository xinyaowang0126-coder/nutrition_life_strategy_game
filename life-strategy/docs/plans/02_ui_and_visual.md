# UI 与视觉工作方案

## 目标

把界面从“信息面板”改造成有校园生活氛围的卡牌游戏：玩家第一眼看到当前时段、场景和可做的选择，需要时再获得详细数据；视觉反馈要清楚，但不持续堆积文字。

## 本轮已落地

- 来源、食物、行动和睡眠选择开始共用 `ChoiceCard` 与 `PhaseChoiceView` 场景，不再分别拼接整套控件。
- 卡牌表面只保留标题、短摘要、价格或标签；详细说明转入鼠标悬停提示。
- 新增短时提示信息流，重要记录出现后会渐隐，不再长期占据右侧面板。
- 右侧信息区改为当天摘要和当前提示，减少常驻日志。
- 新增统一 Godot Theme 资源，初步统一按钮、颜色、字号和交互状态。
- 结束页置于可滚动容器中，降低小屏内容溢出风险。
- 主界面加入紧凑和竖屏适配逻辑，左右栏在空间不足时收起。
- 已建立按阶段切换背景色调与背景资源的接口。
- 已用 ImageGen 分别生成并接入食堂、傍晚便利店、雨夜外卖三张独立全幅背景；原图尺寸、四边和中心 UI 安全区均已人工检查，没有使用图集裁切。
- 已压缩地点卡高度和表面文案，1920×1080 下的 2×2 地点选择可完整显示，底栏不再遮住第二行按钮。
- 已通过 Godot MCP 检查 1920×1080、1366×768 与 390×844：桌面三栏可读，竖屏会收起侧栏并切为单列卡牌，主要按钮可操作。

## 仍需后续

### P0：完成多尺寸与输入验收

- 验证悬停、选中、不可用、钱不足、库存耗尽五种状态的视觉差异。
- 为键盘焦点和手柄焦点补上明显描边；触屏没有悬停时，用轻触或信息按钮展开详情。
- 补做 1280×720、900×900 与超窄屏的边界验收，并把截图纳入发布检查。

### P1：建立完整视觉语言

- 色彩：清晨偏奶油黄，白天偏清爽青绿，夜晚偏深蓝紫；危险信息用低饱和暖红，避免整屏高警示色。
- 材质：使用轻微纸张、水彩和柔和阴影，保持现有插画的手绘气质。
- 层级：背景负责氛围，卡牌负责选择，顶部只显示时间与关键资源，详细规则按需出现。
- 动效：卡牌悬停轻抬、确认轻微回弹、数值变化短暂闪色、时段切换柔和淡入淡出；单次动画控制在约 0.15–0.4 秒。
- 图标：为现金、余力、身体负担、复习和库存建立统一的小图标，逐步替代重复标签文字。

### P1：继续扩充场景素材

- 在现有食堂、便利店、外卖/宿舍夜晚背景之外，补充宿舍清晨、校园下午和结局专用画面。
- 背景中央约一半区域保持低细节和低对比，主要物件放在边缘，为卡牌和面板留出阅读空间。
- 素材不得包含文字、水印或需要精确裁边的 UI 元素；导入后保留原始构图，再由 Godot 的展开模式适配屏幕。
- 对所有新图检查四边、主体完整性、透明通道和不同宽高比下的裁切结果。

### P2：增强游戏反馈

- 每餐结算加入简短的餐盘/收据反馈，集中呈现花费和三项关键变化。
- 一天结束时只显示 2–3 条值得记住的总结，而不是完整流水账。
- 在第七天与结局页加入一周轨迹的小型可视化，例如餐费、跳餐和复习走势。

## 迭代阶段

1. 可用性阶段：无遮挡、可点击、状态清楚、多尺寸可操作。
2. 风格阶段：背景、色彩、字体、卡牌和图标形成统一语言。
3. 打磨阶段：补足动效、音效触发点与一周复盘表现。

## 验收标准

- 默认界面无需阅读长段文字，就能看懂当前时段、资源和下一步操作。
- 所有详细规则可通过悬停、焦点或触屏展开访问，不因减少常驻文本而丢失信息。
- 任何支持尺寸中，主要按钮不被遮挡，结束页和长说明可滚动。
- 新背景均为独立 16:9 文件，四边与主体无误裁，文字区域保持足够对比度。
- 卡牌的可用、不可用、悬停、选中和确认反馈无需依赖文字也能区分。
- 完成一轮流程时，没有提示长期堆积或突然跳动布局。
