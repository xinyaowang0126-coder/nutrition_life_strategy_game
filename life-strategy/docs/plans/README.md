# 后续工作方案总览

这组文档把当前项目的后续工作拆成四条可以独立验收的路线。每份方案都区分了“本轮已落地”和“后续建议”，避免把设想误当成已完成功能。

## 当前判断

项目已经从“可运行的脚本原型”进入“可继续迭代的游戏原型”阶段：七天循环、用餐/行动/睡眠与结局流程能够工作，餐前来源选择已接入，核心结算开始服务化，主界面也开始由可复用 Godot 场景组成。接下来最重要的不是继续堆功能，而是把一次完整游玩打磨得清楚、稳定、有反馈。

## 四条工作路线

1. [玩法与餐前来源](./01_gameplay_and_meal_sources.md)：完善不同来源的策略差异、卡池与经济闭环。
2. [UI 与视觉](./02_ui_and_visual.md)：减少常驻文字，建立场景氛围、卡牌反馈和多分辨率布局。
3. [文案与叙事](./03_copy_and_narrative.md)：统一自然、克制的校园生活口吻，并让数值变化具有生活语境。
4. [Godot 场景架构与测试](./04_godot_scene_architecture_and_testing.md)：继续拆分控制器、数据和视图，用场景、资源、信号与自动化测试支撑迭代。

## 建议执行顺序

- 第一阶段：完成餐前来源闭环与关键结算回归测试。
- 第二阶段：完成主流程 UI、背景素材和卡牌交互的视觉验收。
- 第三阶段：通读并重写全部玩家可见文案，补齐事件与结局差异。
- 第四阶段：继续拆分 `GameRoot`，补充跨场景集成测试和多尺寸验收。

四条路线可以并行制作，但每个可发布版本都应同时通过：一轮完整流程、数值一致性、无报错、多尺寸可操作、玩家可见文本复核。
